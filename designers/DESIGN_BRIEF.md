# 虚拟社区（Virtual Community）— AI 设计师简报

> 本文档面向 AI 设计师，帮助理解项目背景、功能结构和交互逻辑，以便给出更好的 UI/UX 设计方案。

---

## 一、项目背景与定位

**虚拟社区**是一个基于真实调研数据的虚拟舆论模拟平台。

用户提出一个问题（如社会话题、政策、观点），系统中 20 个基于真实价值观调研数据构建的"虚拟网民"会模拟真实社交媒体上的行为——路过、点赞、发表回答、互相回复。用户也可以直接与虚拟人对话，虚拟人会保持角色一致性作出回应。

**核心价值主张**：
- 不是随机 AI，每个虚拟人都有数据基础（5850 名真实受访者的价值观调研 → K-means 聚类 → 20 个有人物故事的虚拟人）
- 不是静态结果，是动态过程（用户实时看到每个虚拟人"经过"并做出反应）
- 不是信息茧房，20 人覆盖 6 种价值观群体，有支持也有反对

**目标用户**：需要快速了解不同价值观群体对某个话题看法的研究者、策划者、内容创作者。

---

## 二、虚拟人设定

系统中有 **20 个虚拟人**，来自真实调研数据聚类出的 **6 类价值观群体**：

| 代码 | 标签 | 人数 | 核心特征 |
|------|------|------|----------|
| C0 | 现代进步派 | 3 人 | 反传统性别观、反物质主义、强调个人自由 |
| C1 | 乐观爱国派 | 4 人 | 高国家认同、社会乐观、重视传统 |
| C2 | 传统全能派 | 3 人 | 全面保守：性别分工、传宗接代、孝道 |
| C3 | 开放理性派 | 2 人 | LGBT 友好、批判权威、理性评估社会 |
| C4 | 沉默中间派 | 5 人 | 无强烈立场、生活满足感略低（最大群体） |
| C5 | 功利强权派 | 2 人 | 金钱至上、性别保守、社会不满 |

每个虚拟人有：年龄、性别、城市、职业、学历、150-200 字的人物故事。

**示例虚拟人**（C4·沉默中间派）：
> 张志远，男，41 岁，郑州，制造业工人，初中学历。在流水线上工作了二十年，每天重复相似的动作。不太关心政治或社会议题，觉得这些离自己太远。最在意的是工厂不倒闭、儿子能继续上学。偶尔刷短视频，见到太极端的评论就划走……

## 三、主要功能模块

### 3.1 提问（/new）

用户每天最多提 3 个问题（每日限额机制）。

- 大文本框输入问题
- 显示今日剩余次数
- 未登录时有引导登录提示
- 提交后跳转到问题详情页，系统开始模拟

### 3.2 问题详情页（/q/[id]）

**核心页面**，三列布局：

```
[左侧]          [中间]                  [右侧]
历史问题列表     回答列表                实时动态日志
                + 用户回复表单           统计面板
```

**左侧 - 侧边栏 (Sidebar)**
- 用户历史问题列表（按时间倒序）
- 状态标签：等待中 / 进行中 / 已完成
- 新建问题按钮
- 底部用户信息 + 登出

**中间 - 回答列表 (AnswerList)**
- 显示虚拟人的回答（Top 9 热门 + 1 条随机探索）
- 每条回答显示：虚拟人头像/名字（Hover 展示人物卡片）、立场标签（支持/中立/反对）、点赞数、回答内容
- 每条回答下方可展开"+ 回复"输入框
- 用户回复后，虚拟人会针对性生成回应（嵌套显示）
- 回复区分：用户发言（"我"）vs 虚拟人发言（名字+角色）

**右侧 - 统计面板 (Dashboard)**
- 公开/私密切换（公开后生成专属 slug URL）
- 处理进度条（已看到/总人数）
- 立场分布：支持 / 反对 / 中立 数量柱状图
- 行为明细：看到多少人、路过多少、点赞问题、回答、点赞回答、回复

**右侧 - 实时动态日志 (ActivityLog)**
- 每个虚拟人完成行为后实时推送一条日志
- 格式：[虚拟人卡] + [动作标签]（发表回答 / 路过 / 点赞 / 回复）
- 不同动作用不同颜色标签
- 自动滚动到最新

### 3.3 公开问题广场（/）首页

- 所有公开问题的 Feed 流
- 每张卡片显示：问题文字、支持/反对/中立比例条
- 响应式网格布局
- 顶部有 Canvas 动画背景（60 个移动节点 + 连接线，模拟网络效果）

### 3.4 公开只读页（/s/[slug]）

- 同问题详情页，但为只读模式
- 隐藏"+ 回复"输入框
- 隐藏公开/私密切换
- 任何人可访问（无需登录）

### 3.5 原理介绍页（/how-it-works）

- 介绍系统如何工作：调研数据 → 聚类 → 虚拟人 → 模拟过程
- 配有图表（聚类热力图、雷达图等）

## 四、系统工作流程

```
用户提问
    ↓
LLM 分析问题 → 生成 20 维价值观向量
    ↓
20 个虚拟人逐个处理（串行）：
    - 计算"与问题的匹配度"
    - 根据匹配度概率采样行为：
        路过(55%) / 点赞问题(20%) / 发表回答(15%)
        + 点赞他人回答(7%) / 回复他人回答(3%)
    - 高匹配度时：回答概率 +20%，路过概率 -25%
    - 如果选择"回答"：LLM 生成回答内容 + 立场判断
    - 如果选择"回复"：LLM 生成回复内容
    - 每步操作实时写库 → 前端实时收到推送
    ↓
全部完成（约 20-40 秒）
    ↓
用户可回复虚拟人 → 虚拟人保持角色生成针对性回应
```

## 五、交互细节与状态

### 加载状态
- 问题提交后：问题状态变为"进行中"，ActivityLog 开始实时刷新
- 每个虚拟人处理完成后，Dashboard 进度条 +1，回答列表实时出现新条目
- 全部完成后：状态变为"已完成"

### 虚拟人 Hover 卡片
- 悬停在虚拟人名字/头像上 300ms 后弹出
- 内容：集群类型、年龄、性别、城市、职业、教育、人物故事

### 用户回复交互
- 点击"+ 回复"按钮，展开输入框（默认折叠）
- 输入后提交，立即显示用户回复
- 虚拟人的回应几秒后出现（异步处理）
- 嵌套层级：回答 → 用户回复 → 虚拟人回应

### 公开分享
- Dashboard 中切换"公开"开关
- 生成唯一 slug（如 `/s/abc123`）
- 显示可复制的分享链接

### 每日限额
- 每用户每天最多 3 个问题
- 提问页显示"今日剩余 X 次"
- 达上限后禁用提交按钮，显示提示

## 六、视觉风格现状

当前实现使用 **Tailwind CSS + shadcn/ui**，风格偏向深色/中性色调。

### 现有视觉元素
- 首页：Canvas 动画网络背景（网络节点 + 连线，体现"社区网络"隐喻）
- 立场标签：支持（绿色）/ 中立（灰色）/ 反对（红色）
- 状态标签：等待中（黄色）/ 进行中（蓝色）/ 已完成（绿色）
- ActivityLog 动作标签：不同颜色区分 5 种动作类型
- 进度条：线性进度（已处理/总人数）

### 现有布局
- 桌面端三列布局：侧边栏(w-56) + 主内容区 + 右侧面板
- 首页响应式网格卡片
- 暂无移动端适配

## 七、当前痛点与设计挑战

以下是开发过程中识别到的 UI/UX 问题，希望设计师重点关注：

1. **虚拟人形象感不足**：目前虚拟人只有文字名称，缺乏视觉身份（头像/图标/颜色），在密集列表中难以区分

2. **等待过程体验单薄**：20 个虚拟人串行处理需要 20-40 秒，目前只有 ActivityLog 滚动，缺乏吸引人的等待状态设计

3. **信息密度过高**：问题详情页三列信息量大，特别是右侧 Dashboard + ActivityLog 叠加，视觉层次不清晰

4. **回复树层级视觉**：嵌套回复（回答→用户回复→虚拟人回应）的层级关系在视觉上不够直观

5. **首页缺乏吸引力**：FeedCard 目前只有文字 + 比例条，缺乏视觉兴趣点

6. **公开广场与个人空间区分不明显**：首页（公开广场）和 /q/[id]（个人问题）的场景差异没有通过视觉明确传达

7. **How It Works 页面平淡**：展示系统原理很重要，但目前偏向技术文档风格，缺乏引导性的视觉叙事

## 八、技术约束

- **框架**：Next.js App Router + Tailwind CSS v4 + shadcn/ui
- **实时推送**：Convex reactive query（无需 WebSocket 配置，数据变更自动推送到前端）
- **动画**：目前使用原生 Canvas（NetworkHero），可考虑引入 Framer Motion 或 CSS 动画
- **图标**：目前暂未统一使用图标库（可引入 Lucide、Heroicons 等）
- **字体**：使用系统默认字体（可引入自定义字体）

---

## 九、设计参考方向（供参考，非强制）

- 产品定位偏向**研究工具 + 社交实验**，可参考 Twitter/微博 的内容流设计，但带有更强的"分析/洞察"感
- 虚拟人群体感可借鉴**数据可视化**风格（如 The Pudding 的数据新闻）
- 实时过程体验可参考**游戏 loading 页**或**直播弹幕**场景，强调"正在发生"的临场感
- 整体色调：目前深色中性，可探索更有个性的品牌色
